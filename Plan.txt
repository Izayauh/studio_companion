# Project Specification: Text-to-MIDI Chord Generator

## 1. Project Overview
This script will function as a modular text-to-MIDI bridge for a larger AI assistant system (Jarvis Ableton). The goal is to take high-level, emotionally-driven chord progressions generated by an LLM (formatted as a JSON array of standard chord names) and translate them into a playable, drag-and-drop `.mid` file for Ableton. 

The output must support complex R&B/Pop voicings (e.g., 9ths, 11ths, passing chords) reminiscent of artists like Frank Ocean or Jon Bellion, which is why a dedicated music theory library is required to calculate the intervals before writing the MIDI file.

## 2. Tech Stack
* **Language:** Python 3.x
* **The Brain (Theory & Voicing):** `mingus` (Specifically `mingus.core.chords` to parse string names like "Fm9" into note arrays).
* **The Hands (File Creation):** `midiutil` (Specifically `MIDIFile` to write the note data to a physical file).
* **Data Input:** `json` (Standard library for parsing the LLM output).

## 3. System Architecture & Workflow
1.  **Input:** The script will accept a `.json` file (or a direct JSON string argument) containing an array of chord names.
2.  **Processing (Mingus):** The script loops through the array. It uses `mingus` to translate each chord string into its constituent notes. 
3.  **MIDI Mapping:** The script maps those notes to standard MIDI integer values, assigning a default octave (e.g., Octave 4) to ensure the chords sit in the mid-range of a piano VST.
4.  **Output (Midiutil):** The script writes the mapped notes into a standard 4/4 time signature (one chord per whole note/bar) and exports `output_loop.mid` to the local directory.

## 4. Input Data Format (From LLM)
The Python script must be built to parse the following exact JSON structure. (This is the strict output format the LLM has been prompted to provide).

```json
{
  "chords": ["Fm9", "Dbmaj7", "Eb6", "C7b9"]
}
